我们的代码里使用了 localStorage（浏览器本地存储技术）。这相当于在你的手机浏览器里借用了一块“小 U 盘”，专门用来存放你这个决定器的数据。你今天关掉网页、杀掉浏览器后台，明天再打开，数据和图片都会乖乖待在那里。

但是，既然是“本地存储”，在手机上使用时就有几个容易导致数据丢失的雷区，你需要特别注意：

清理手机缓存/浏览器数据：如果你在手机设置里清理了浏览器的缓存，或者使用了清理软件，这些外卖数据就会被一起清空。

无痕模式/隐私模式：如果是在浏览器的“无痕模式”下打开的这个网页，只要一关掉页面，数据就会消失。

微信/QQ 内置浏览器（重点注意）：如果你是把链接发在微信里，直接在微信里点开用的，微信系统有时会不定时自动清理内置浏览器的缓存文件。

建议：
1.在微信里点开后，点击右上角的三个点，选择“在浏览器打开”（苹果手机用 Safari，安卓用 Chrome 或自带浏览器）。然后在使用系统浏览器时，把它“添加到主屏幕”。这样它就会像一个独立的 App 一样留在你的手机桌面上，数据最稳定！
2.如果换新的设备，可以导出数据，会导出一个.json文件，这时你在新设备导入这个文件，你的记录就回来了。

声明：
1.不互通：你在 Safari 浏览器里存的数据，如果明天换成用 Chrome 浏览器打开，是看不到的。数据是跟着具体哪个浏览器走的。

2.浏览器给每个网页分配的 localStorage 本地存储最大容量，业界标准统一是 5MB（部分浏览器可能会给到 10MB，但我们在手机上通常按最保守的 5MB 来算）。
5MB 听起来似乎非常小（现在手机随便拍一张原图都要 5MB 以上了），但在加入了前端极限压缩代码之后，这 5MB 其实相当耐用。
粗劣计算：文字信息（店名、备注、分类等）占用的空间几乎可以忽略不计。经过 Canvas 压缩后，一张长宽在 600 像素以内的照片，转换成代码（Base64 字符串）后的大小大约在 30KB 到 50KB 之间。
如果你每次只传 1 张图：5MB 大约能存 100 到 150 个 外卖菜品。
如果每次都传满 5 张图：那每个菜品就要占大概 200KB，存大概 20 到 25 个 菜品就会把浏览器的空间彻底塞满。
